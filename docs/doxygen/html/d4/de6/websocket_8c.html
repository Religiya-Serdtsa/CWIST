◆ cwist_websocket_receive()

cwist_ws_frame * cwist_websocket_receive ( cwist_websocket * ws)

Receive the next WebSocket frame from a connected client.

Receive a frame, blocking until data arrives or the socket closes.

Client frames are required to be masked by RFC 6455, so unmasked payloads are rejected. Payload data is copied into a heap buffer and null-terminated for convenience when the caller interprets a text frame as a C string.

Parameters
wsWebSocket connection wrapper returned by cwist_websocket_upgrade().
Returns
Newly allocated frame, or NULL when the connection is closed or invalid.

int client_fd )

Upgrade an HTTP request to a WebSocket connection.

Upgrade a standard HTTP request to a WebSocket connection.

The function validates the required upgrade headers, computes the Sec-WebSocket-Accept response, emits the 101 Switching Protocols reply, and returns a lightweight wrapper around the already-open client socket.

Parameters
reqParsed HTTP request that requested the upgrade.
client_fdAccepted client socket descriptor.
Returns
WebSocket wrapper on success, or NULL when the handshake fails.
