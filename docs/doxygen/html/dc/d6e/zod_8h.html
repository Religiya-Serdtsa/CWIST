Detailed Description

Strict JSON schema validation (Zod-style) for CWIST.

Unlike the self-healing layer, this module rejects data that does not conform to the schema. Use it at trust boundaries (HTTP API inputs, DB writes) where silent coercion is undesirable.

It shares the cwist_schema_t / cwist_schema_field_t types defined in json_heal.h so the same schema descriptor can drive both healing and strict validation.

Usage:

cwist_zod_result_t r = cwist_zod_parse(raw_body, &my_schema, &parsed);
if (!r.valid) {
cwist_zod_print_errors(&r);
// respond 400
}
// use parsed …
cJSON_Delete(parsed);

const cwist_schema_t * schema )

Strictly validate an already-parsed cJSON object against a schema.

Checks:

  • All required fields are present.
  • All fields that appear in the schema have the declared type.

No type coercion is performed; use cwist_json_heal() first if coercion is acceptable.

Parameters
jsoncJSON object (not array / primitive).
schemaSchema descriptor.
Returns
Validation result. result.valid == true iff all checks pass.
