Detailed Description

Schema migration system with version history and rollback.

Each migration is a (version, name, up_sql, down_sql) tuple. The system stores applied migrations in a _cwist_migrations table and executes UP or DOWN SQL inside transactions, so a failed migration leaves the database unchanged.

typedef struct cwist_migration_t cwist_migration_t

A single migration step.

  • version : monotonically increasing integer (e.g. 1, 2, 3 …)
  • name : human-readable label (used in history table)
  • up_sql : SQL executed when migrating forward; may contain multiple statements separated by ';'
  • down_sql: SQL executed when rolling back; NULL means irreversible
